# style
```
[用途]
這是一份給 AI 與程式快速理解的 infographic 範例模板。
保留固定欄位名稱，能提升批次生成時的穩定性。

[解析規則]
- 只能使用 `# style` 與 `# image xx`
- 每個 `# image xx` 代表一張圖
- 區塊內盡量使用固定欄位，不要自由散文
- 若要新增頁面，直接複製一個 `# image xx` 區塊即可

[常用執行方式]
- 跑完整組：
  `python main.py -b infographic-example.md -s 1024x1024 -q low --output-dir output/infographic_example`
- 只跑第 2 張：
  `python main.py -b infographic-example.md --image 2 -s 1024x1024 -q low --output-dir output/infographic_example`
- 跑第 2 到第 3 張：
  `python main.py -b infographic-example.md --range 2-3 -s 1024x1024 -q low --output-dir output/infographic_example`
- 跳過指定張數：
  `python main.py -b infographic-example.md --skip-images 1 -s 1024x1024 -q low --output-dir output/infographic_example`

[尺寸選擇說明]
- `1024x1024`
  用途: 正方形社群輪播
  優點: 生成最快、最省成本
  適合: 草稿、一般資訊圖表、教學輪播

- `1440x1440`
  用途: 高一點解析度的正方形
  優點: 比 1024 更細緻
  適合: 構圖已穩定、想提升品質

- `1024x1536`
  用途: 直式 2:3
  適合: 手機直式圖卡、海報草稿

- `1152x2048`
  用途: 9:16 直式
  適合: 手機全螢幕閱讀圖卡

- `2048x1152`
  用途: 16:9 橫式
  適合: 簡報封面、影片封面、橫式資訊圖

- `auto`
  用途: 交給模型決定
  注意: 若要做系列圖，通常不建議用 `auto`，固定尺寸更穩

[畫質選擇說明]
- `low`
  用途: 測 prompt、看版面、快速草稿

- `medium`
  用途: 第二輪確認、平衡品質與速度

- `high`
  用途: 最終稿
  注意: 成本最高，建議只在 prompt 穩定後使用

[推薦流程]
1. 先用最適合閱讀情境的尺寸跑 `low`
2. 先確認資訊層級、版面與文字位置
3. 不滿意的頁面用 `--image` 或 `--range` 單獨重跑
4. 最後再升到 `medium` 或 `high`

[全系列共用設定]
類型: infographic
版面: 1024x1024, 正方形
風格: 乾淨, 扁平設計, 社群輪播
一致性: 統一配色, 統一字體風格, 統一圖示語言, 統一卡片樣式
閱讀感: 留白清楚, 標題層級明確, 資訊容易快速掃讀

[建議欄位寫法]
- 主題: 這張圖在講什麼
- 目的: 這張圖希望讀者看完得到什麼
- 版面: 要怎麼排
- 元素: 要有哪些圖示 / 卡片 / 區塊
- 強調: 哪裡要最醒目
- 避免: 不希望模型做出什麼

[注意]
- 用短句
- 一行一個重點
- 先寫資訊結構，再寫美術形容
- 避免只寫「好看、簡潔、清楚」這種空泛詞，最好寫清楚怎麼清楚
- 若是系列圖，建議固定尺寸與畫質，不要每張都改
- 如果目標是手機閱讀，優先考慮直式尺寸
- 如果目標是社群輪播，正方形通常最穩
```

# image 01_cover
```
[主題]
什麼是過敏

[目的]
做封面頁
讓讀者一眼知道主題是「過敏懶人包」

[版面]
上方: 大標題
中間: 3 個圖示並排
下方: 一句簡短副標

[元素]
圖示1: 鼻子症狀
圖示2: 皮膚症狀
圖示3: 眼睛症狀

[強調]
主標題最大
三個圖示要簡單、辨識度高

[避免]
不要做成海報插畫
不要背景過滿
不要圖示風格不一致
```

# image 02_triggers
```
[主題]
常見誘因

[目的]
列出最常見的 4 種過敏誘因
讓讀者快速理解生活中常見來源

[版面]
四格卡片
2 x 2 排列
每格: 圖示 + 名稱

[元素]
卡片1: 食物
卡片2: 塵蟎
卡片3: 花粉
卡片4: 寵物毛髮

[強調]
四格大小一致
名稱清楚可讀
圖示比裝飾更重要

[避免]
不要塞太多說明文字
不要做成複雜表格
不要每格用不同設計語言
```

# image 03_warning_signs
```
[主題]
何時該看醫生

[目的]
提醒讀者注意 3 個需要就醫的警訊

[版面]
垂直三段式
每段一個警訊卡片
底部可有總結提醒

[元素]
警訊1: 症狀持續惡化
警訊2: 呼吸不舒服
警訊3: 影響日常生活

[強調]
警訊標題要醒目
可用紅色做小範圍重點提示
但整體仍保持同系列視覺

[避免]
不要整頁變成大面積警告紅
不要做出醫療恐嚇感
不要資訊層級混亂
```
