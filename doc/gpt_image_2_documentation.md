
# GPT Image 2 官方文件精華總結

**gpt-image-2** 官方文件濃縮成最核心的精華總結。這份說明主要可以拆解為「存取模式」、「核心功能」、「自訂選項」與「使用限制」四大板塊：

## 1. 兩種 API 存取模式

根據你的應用場景，OpenAI 提供了兩條路徑來使用圖像模型：

* **Image API (單次生成)**：最直接的「投幣機」模式。適合一問一答的單次產圖或編輯，程式碼最簡單。
* **Responses API (多輪對話)**：適合用來做聊天機器人（如你正在規劃的架構）。搭配 `gpt-5.5` 等文字模型並啟用 `image_generation` 工具，系統可以記住歷史對話，透過傳遞「ID」來對同一張圖片進行反覆、迭代的修改。

## 2. 核心產圖與編輯功能

* **從零生成 (Generations)**：單純透過文字提示詞 (Prompt) 生成圖片，API 預設回傳 Base64 格式的圖片資料。
* **參考圖合成**：可以上傳多張現有圖片作為「參考」，讓 AI 依照這些素材生成一張全新的情境圖（例如把幾個獨立的商品合成一個禮物籃）。
* **遮罩編輯 (Masking/Inpainting)**：提供原圖與一張「遮罩圖」，讓 AI 只修改指定的區域。**注意：遮罩圖必須包含 Alpha 通道（透明度）**。
* **流式傳輸 (Streaming)**：支援設定 `partial_images` (0-3 張)，在最終圖片完成前，先回傳幾張低解析度的半成品預覽圖，提升用戶等待時的體驗。

## 3. 圖片輸出與自訂選項

* **尺寸彈性 (Size)**：支援非常自由的解析度（最大邊 3840px，比例不超過 3:1，須為 16px 的倍數），正方形圖片生成速度最快。
* **畫質控制 (Quality)**：可設定 `low`、`medium`、`high` 或自動 (`auto`)。建議測試或草稿期使用 `low` 以節省時間與成本。
* **檔案格式 (Format)**：預設為 `png`，但也支援 `jpeg` 與 `webp`。如果追求速度，選擇 `jpeg`/`webp` 並搭配 `output_compression` 參數設定壓縮率，可以有效降低延遲。
* **⚠ 透明背景限制**：最新的 `gpt-image-2` **不支援**透明背景 (`background: "transparent"`)。
* **⚠ 輸入保真度 (Fidelity)**：`gpt-image-2` 預設將所有輸入的參考圖片視為「高保真 (High Fidelity)」，無法調低，這意味著編輯圖片時會消耗較多的輸入 tokens。

## 4. 效能、限制與成本

* **處理時間**：複雜的指令最多可能需要長達 2 分鐘的時間來生成。
* **模型弱點**：雖然已經很強大，但在「精準渲染圖片中的文字」、「維持多張圖的角色一致性」以及「極度精確的版面空間控制」上，仍有一定的挑戰。
* **成本計算**：總花費 = 輸入的文字 Tokens + 輸入的參考圖片 Tokens (如果有的話) + 輸出的圖片 Tokens (由你設定的尺寸和畫質決定)。如果有開啟流式傳輸的預覽圖，每張預覽圖會額外加收 100 tokens。
* **內容審查**：可透過 `moderation` 參數設定審查嚴格度（預設為 `auto`，也可設為較寬鬆的 `low`）。

## 5. 前置作業要求

* **組織驗證**：開發者必須在控制台完成 Organization Verification (身份/組織驗證) 才能獲取 `gpt-image-2` 等受保護模型的使用權限。
